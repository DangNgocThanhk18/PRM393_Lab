# Exercise 12.1 — Optimize List Rebuilds

## Mục tiêu
Giảm thiểu việc rebuild không cần thiết khi thêm/sửa/xóa task trong danh sách.

## Các vấn đề phát hiện

### Vấn đề 1: Inline widget trong ListView.builder
- **Mô tả**: Task được xây dựng trực tiếp trong itemBuilder
- **Tác động**: Mỗi lần state thay đổi, toàn bộ danh sách bị rebuild
- **Giải pháp**: Tách thành widget riêng TaskTile

### Vấn đề 2: Thiếu key cho widget
- **Mô tả**: Không có key để Flutter nhận diện widget
- **Tác động**: Flutter không biết widget nào thay đổi, rebuild toàn bộ
- **Giải pháp**: Thêm ValueKey(task.id)

### Vấn đề 3: Watch toàn bộ state
- **Mô tả**: ref.watch(taskListProvider) watch toàn bộ state
- **Tác động**: Mọi thay đổi đều trigger rebuild toàn bộ UI
- **Giải pháp**: Sử dụng Consumer chỉ watch phần cần thiết

## Giải pháp đã áp dụng

### 1. Tách TaskTile thành widget riêng

```dart
class TaskTile extends ConsumerWidget {
  final Task task;
  
  const TaskTile({super.key, required this.task});
  
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    return ListTile(
      key: ValueKey(task.id),
      leading: Checkbox(
        value: task.isCompleted,
        onChanged: (_) {
          ref.read(taskListProvider.notifier).toggleTask(task.id);
        },
      ),
      title: Text(task.title),
      onTap: () => Navigator.push(...),
    );
  }
}
```
### 2. Sử dụng Consumer pattern
```dart
Expanded(
child: Consumer(
builder: (context, ref, child) {
final tasks = ref.watch(taskListProvider);
// Chỉ rebuild khi tasks thay đổi
return ListView.builder(...);
},
),
)
```
### 3. Thêm const constructor
```dart
const TaskTile({super.key, required this.task});
```
